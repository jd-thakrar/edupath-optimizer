How It Works

Six-layer AI architecture for intelligent decision support

1

Feature Engineering Layer

Extracts 22 intelligent features: attendance trends, performance volatility, engagement scores, semester load, historical patterns. Temporal features capture trajectory, not just current state.

2

Risk Prediction Model

Gradient Boosting classifier learns patterns from data. Calibrated for accurate probabilities. Returns: failure_probability, confidence, risk_level. No hardcoded thresholds.

3

Counterfactual Simulation Engine

Simulates "what-if" scenarios: What if attendance improves 15%? What if marks increase by 5? Ranks interventions by effectiveness score (risk reduction per effort unit).

4

Knowledge Dependency Graph

Graph neural network of course prerequisites. Propagates risk forward: failing Calculus I affects Calculus II (90% dependency), Physics (75%), Engineering Math (80%).

5

Explanation Layer (Gemini)

Gemini API translates ML outputs to natural language. Explains uncertainty, justifies recommendations, provides context-aware insights. Gemini does NOT make decisions—only explains them.

6

Continuous Learning

Model retrains on new data. Improves predictions over time. Adapts to institutional patterns. Admin dashboard tracks model performance and data quality.
